# Watch List - Совместное ведение списков просмотра

## Краткое описание

Веб-приложение для совместного ведения списков (досок) с элементами: фильмы, сериалы, книги, игры. Участники добавляют элементы в свои списки, система формирует общий список с чередованием элементов по кругу (ABAB, ABCABC).

## Технологический стек

### Frontend

- **React 19** - основной фреймворк
- **Vite** - сборка и dev-сервер
- **Mantine UI** - компонентная библиотека (v8.3.16)
- **React Router DOM** - роутинг (v7.13.1)
- **@dnd-kit** - drag-and-drop функциональность
- **@convex-dev/auth** - аутентификация

### Backend

- **Convex** - backend-as-a-service (v1.33.0)
- **Convex Auth** - система аутентификации
- **Convex Schema** - типобезопасная схема данных

### Инструменты

- **ESLint + Prettier** - линтинг и форматирование
- **GitHub Pages** - деплой

### Дополнительная документация

- @https://docs.convex.dev/llms-full.txt

## Сущности данных (Convex Schema)

### users

- Автоматическая таблица от Convex Auth
- Дополнительно: `name`, `email`

### boards

- `_id`, `name`, `createdBy`, `createdAt`, `isActive`, `emoji`

### boardMembers

- `_id`, `boardId`, `userId`, `role`, `joinedAt`
- Роли: `creator`, `moderator`, `member`

### items

- `_id`, `boardId`, `userId`, `title`, `totalEpisodes`, `currentEpisode`, `url`, `order`, `createdAt`, `updatedAt`, `isHidden`

### history

- `_id`, `boardId`, `userId`, `itemId`, `action`, `details`, `timestamp`
- Действия: `create`, `update`, `delete`, `complete`, `uncomplete`

## Основные функции

### Аутентификация

- Регистрация и вход через Convex Auth
- Автоматическое создание профиля пользователя
- Защита маршрутов

### Доски

- Создание досок с эмодзи
- Приглашение участников по email
- Ролевая модель доступа
- Удаление досок (только создателем)

### Элементы списков

- Добавление элементов (название, эпизоды, ссылка)
- Редактирование любыми участниками
- Отметка "просмотрено" (установка currentEpisode = totalEpisodes)
- Скрытие элементов (soft delete)
- Drag-and-drop сортировка (только свои элементы)

### Общий список

- Автоматическое чередование элементов всех участников
- Алгоритм round-robin (ABAB, ABCABC)
- Отображение прогресса в процентах

### История

- Полная аудитория всех действий
- Отображение в хронологическом порядке
- Фильтрация по типам действий
